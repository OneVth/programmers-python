## 🎯 핵심 학습 내용

### 1. 등차수열 인덱스 슬라이싱 패턴 (문제 #181903, #181904)

```python
# qr code: 나머지가 r인 인덱스
code[r::q]          # start=r, step=q

# 세로 읽기: c번째 열 (1-indexed)
my_string[c-1::m]   # start=c-1, step=m
```

- 두 문제는 표현만 다를 뿐 **완전히 동일한 패턴** — 문제를 수식으로 치환하면 구조가 보인다
- `[start::step]`은 `range(start, len, step)`과 수학적으로 동일한 인덱스 집합을 선택

### 2. `[::-1]`의 내부 구현

```python
# 세 단계로 동작
# 1. slice(None, None, -1) 객체 생성
# 2. start = len-1, stop = -(len+1), step = -1 계산
# 3. 새 버퍼 할당 후 역순 memcpy
```

| 대상 | `[::-1]` | 복사 여부 | 복잡도 |
|------|----------|-----------|--------|
| `str` / `list` | 새 객체 생성 | O | O(n) |
| `list.reverse()` | in-place | X | O(n) |
| NumPy array | stride view | X | **O(1)** |

- NumPy는 실제 데이터를 복사하지 않고 **stride(보폭)를 음수로 설정**한 view만 반환

### 3. 구간 역전 패턴 (문제 #181905)

```python
# 인덱스 s~e 구간만 뒤집기
my_string[:s] + my_string[s:e+1][::-1] + my_string[e+1:]
```

- off-by-one 주의: `[s:e+1]`에서 `+1` 빠뜨리기 쉬움 — 슬라이스는 end 인덱스 미포함
- 공간복잡도 O(n): 슬라이싱 3회 + 연결 1회 = 총 4개 문자열 객체 생성

### 4. `startswith()` vs `find()` 트릭 (문제 #181906)

```python
# ❌ find() 트릭 — 의도 불명확
return 1 if not my_string.find(is_prefix) else 0
# find()가 0 반환 → not 0 = True → 1 (접두사)
# find()가 -1 반환 → not -1 = False → 0 (없음)
# find()가 양수 반환 → not 양수 = False → 0 (중간 발견)

# ✅ startswith() — 명확하고 빠름
return int(my_string.startswith(is_prefix))
```

- `startswith()` / `endswith()`는 접두사/접미사 길이 k만큼만 비교 → O(k)
- `find()`는 전체 문자열을 탐색할 수 있어 최악 O(n)

### 5. 접두사/접미사 3부작 구조 (문제 #181906~181908)

```
181906 접두사인지 확인하기 → startswith()
181907 문자열의 앞의 n글자 → [:n]  (접두사의 정의 자체)
181908 접미사인지 확인하기 → endswith()
```

- 같은 개념을 다양한 각도로 연습하도록 설계된 시리즈
- `bool`은 `int`의 서브클래스 — `True == 1`, `False == 0` 성립, 하지만 타입 힌트 `-> int`를 위해 `int()` 변환 명시 권장

## 📝 오늘의 패턴 요약

오늘 6문제 모두 **Python 슬라이싱의 다양한 활용**이 핵심이었다.

```
[r::q]          등차수열 필터링
[:s] + [s:e+1][::-1] + [e+1:]  구간 역전
[:n]            접두사 추출
[-n:]           접미사 추출
```

슬라이싱 하나로 2D 배열, 구간 조작, 접두/접미사를 모두 처리할 수 있다는 점에서 Python의 표현력이 돋보이는 문제 세트였다.

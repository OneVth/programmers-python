## 🎯 핵심 학습 내용

### 1. 문자열 슬라이싱 3분할 패턴 — 문자열 겹쳐쓰기 (#181943)

```python
# s 위치부터 overwrite_string 길이만큼 교체
my_string[:s] + overwrite_string + my_string[s + len(overwrite_string):]
```

- 파이썬 문자열은 불변(immutable) → 인덱스로 직접 수정 불가, 슬라이싱으로 잘라 이어붙임
- `s + len(overwrite_string)`이 "덮어쓴 끝 인덱스"를 정확히 계산
- 제한 조건 `0 ≤ s ≤ len(my_string) - len(overwrite_string)` 덕분에 범위 초과 예외처리 불필요
  → **제한 조건을 신뢰해 불필요한 검사를 생략하는 것이 코딩테스트 관례**

### 2. print 출력 문제를 runner 호환으로 변환 — 홀짝/문자열 돌리기 등

이 시리즈의 여러 문제는 `print` 출력 형식이지만, `runner.py`가 반환값을 비교하므로 **문자열 반환**으로 구현.

```python
# 홀짝 구분하기 (#181944)
def solution_v1(n: int) -> str:
    if n % 2 == 0:
        return f"{n} is even"
    else:
        return f"{n} is odd"

# 문자열 돌리기 (#181945) — "\n"으로 이어붙이면 출력과 동일한 효과
def solution_v1(str: str) -> str:
    return "\n".join(str)
```

### 3. 문자열은 iterable — 문자열 돌리기 (#181945)

```python
"\n".join("abcde")   # → "a\nb\nc\nd\ne"
# list("abcde") 나 [c for c in "abcde"] 변환 없이 바로 사용 가능
```

- 파이썬에서 문자열은 그 자체로 iterable → `for`, `join`, `zip`, `enumerate` 등에 바로 전달 가능
- `str`을 파라미터 이름으로 쓰면 내장 타입 `str`을 **shadowing** → 실무에서는 `s` 또는 `string` 권장
  (이 문제는 프로그래머스 지정 이름이라 예외)

### 4. 문자열 연결 횟수와 효율 — 문자열 붙여서 출력하기 (#181946)

```python
str1 + str2               # 단순 연결: O(n), 한 번만 연결할 때 충분
"".join([str1, str2])     # join 방식: 동일 결과, 관용적
```

- **루프 안에서 `+` 반복 연결은 O(n²)** → 매번 새 객체 생성
- `"".join(iterable)` 은 전체 길이를 미리 계산해 **메모리를 한 번만 할당** → O(n)
- 단 한 번의 연결은 `+`와 `join` 모두 O(n)으로 차이 없음

### 5. f-string 안 표현식 — 덧셈식 출력하기 (#181947)

```python
f"{a} + {b} = {a + b}"   # {} 안에 산술 표현식 직접 사용 가능
f"{a:.2f}"                # 포맷 지정자와 함께 사용
f"{'even' if n % 2 == 0 else 'odd'}"  # 조건식도 가능
```

- f-string의 `{}` 안은 **임의의 파이썬 표현식**을 받음 (변수, 연산, 메서드 호출, 조건식)
- 별도 변수 선언 없이 인라인으로 계산 → 코드 간결화

### 6. raw string vs 이스케이프 — 특수문자 출력하기 (#181948)

```python
# 출력 목표: !@#$%^&*(\'"<>?:;

# v1: raw string — 백슬래시가 literal, \'는 따옴표 종료를 막기 위해 필요
r'!@#$%^&*(\'"<>?:;'

# v2: 일반 문자열 — \\ = \, \' = '
'!@#$%^&*(\\\'"<>?:;'
```

- raw string(`r'...'`)은 `\n`, `\t` 등 이스케이프를 처리하지 않음 → 정규표현식, 파일 경로에 특히 유용
- **raw string의 제약**: 문자열 끝에 홀수 개의 백슬래시를 쓸 수 없음 (`r'\'` 불가)
- `\'`는 raw string에서도 따옴표 종료 방지로 작동하지만 백슬래시는 문자열에 포함됨

## 📝 오늘의 패턴 요약

```
str[:s] + overwrite + str[s+len(overwrite):]   문자열 특정 구간 교체 — 슬라이싱 3분할
"\n".join(str)                                  문자열 각 글자를 줄바꿈으로 구분
"\n".join(iterable)                             문자열은 별도 변환 없이 바로 join 가능
f"{a} + {b} = {a + b}"                         f-string 안에서 표현식 직접 계산
r'...\''                                        raw string — 이스케이프 많은 문자열에 적합
str1 + str2  (단 한 번)                         단순 연결은 + 로 충분, 루프 반복 시엔 join
```

**공통 핵심:** 파이썬 문자열은 **불변 + iterable** 두 가지 성질을 동시에 가짐.
불변이기 때문에 수정 시 슬라이싱으로 재조합하고,
iterable이기 때문에 `join`, `zip`, `for` 등에 리스트 변환 없이 바로 활용할 수 있다.

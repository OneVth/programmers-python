## 🎯 핵심 학습 내용

### 1. bare except를 사용하지 말아야 하는 이유 (문제 #181898)

```python
# ❌ bare except — KeyboardInterrupt, SystemExit까지 잡힘
try:
    return arr.index(1, idx)
except:
    return -1

# ✅ 구체적 예외 명시
try:
    return arr.index(1, idx)
except ValueError:
    return -1
```

- `except:`는 `BaseException` 전체를 잡아 프로그램 종료 시그널까지 삼킨다
- 버그가 숨겨져 디버깅이 극도로 어려워진다
- 모든 주요 린터(flake8 E722, ruff E722)가 경고하는 안티패턴
- **→ 노트 작성: `notes/bare-except-antipattern.md`**

### 2. set의 내부 구조 — 해시 테이블과 O(1) 탐색 (문제 #181900)

```python
# ❌ list in — O(n) 순차 탐색
if i not in indices:  # indices가 list일 때

# ✅ set in — O(1) 해시 탐색
exclude = set(indices)
if i not in exclude:
```

- set은 내부적으로 **해시 테이블**로 구현되어 있다
- `hash(값) % table_size`로 저장 위치를 바로 계산 → O(1) 접근
- 해시 충돌 시 **오픈 어드레싱** 방식으로 다음 슬롯 탐색
- 테이블이 2/3 차면 자동 리사이징 (amortized O(1) 유지)
- **반복적 `in` 연산이 필요하면 list → set 변환이 핵심 최적화**
- **→ 노트 작성: `notes/set-internal-hashtable.md`**

### 3. range()의 step 활용 패턴

```python
# 역순 수열 (문제 #181899)
list(range(start, end - 1, -1))  # stop이 exclusive이므로 -1

# 배수 생성 (문제 #181901)
list(range(k, n + 1, k))  # O(n/k) — 필요한 값만 생성
```

- `range(1, n+1)` 전체를 순회하며 `% k == 0` 체크하는 것보다 step 활용이 훨씬 효율적

### 4. dict.fromkeys()와 삽입 순서 보장 (문제 #181902)

```python
# ord() 매직넘버 없이 깔끔한 카운팅
answer = dict.fromkeys("ABCD...XYZabcd...xyz", 0)
for c in my_string:
    answer[c] += 1
return list(answer.values())  # 삽입 순서대로 반환
```

- Python 3.7+에서 dict는 **삽입 순서를 보장**한다
- `string.ascii_uppercase + string.ascii_lowercase`로 키 문자열 생성 가능

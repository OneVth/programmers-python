## 🎯 핵심 학습 내용

### 1. Digit Sum Property (문제 #181914)

```python
# n % 9 == sum(각 자리 숫자) % 9
return sum(int(c) for c in number) % 9
```

- **수학적 근거**: 10 ≡ 1 (mod 9) 이므로 10^k ≡ 1 (mod 9) → 각 자리 숫자의 합으로 대체 가능
- 같은 이유로 **3의 배수 판별**에도 동일하게 적용됨
- `number`가 최대 100,000자리 → `int(number) % 9`도 Python에서는 작동하지만, digit sum이 이 성질을 직접 활용하는 풀이
- Generator in `sum()`: 중간 리스트 없이 O(1) 공간

### 2. 인덱스 기반 문자열 조합 (문제 #181915)

```python
return "".join(my_string[i] for i in index_list)
```

- `join` + generator: 중간 리스트 없이 O(N) 공간 효율적 처리
- **인덱스 중복 허용**: 같은 인덱스가 여러 번 등장 가능 (집합이 아닌 순서 있는 배열)
- `+=` 문자열 누적은 매 반복마다 새 객체 생성 → O(N²), `join`은 O(N)

### 3. Counter로 케이스 분기 (문제 #181916)

```python
from collections import Counter

counter = Counter((a, b, c, d))
len_counter = len(counter)          # 고유값 개수 → 케이스 구분

# len=1: 모두 같음     → 1111 * p
# len=2, max=3: 3+1   → (10*p + q)²
# len=2, max=2: 2+2   → (p+q) * |p-q|
# len=3: 2+1+1        → q * r  (most_common()[1][0] * most_common()[2][0])
# len=4: 모두 다름    → min(nums)
```

- `most_common()`은 빈도 내림차순 → `len=3`에서 쌍(count=2)이 `[0]`, 단독값들이 `[1]`, `[2]`에 위치
- 입력이 항상 4개로 고정이므로 Counter 연산은 실질적으로 **O(1)**

## 💬 오늘의 Q&A

**Q. 코딩테스트에서 `Counter` 사용 가능한가?**
- ✅ 가능. `collections.Counter`는 Python 표준 라이브러리 → 온라인 저지 전부 허용
- ❌ 불가 → `numpy`, `pandas` 등 외부 설치 필요한 패키지

**Q. VS Code 같은 에디터 사용 가능한가?**

| 유형 | 에디터 허용 |
|------|------------|
| 온라인 과제형 (사전 과제 등) | ✅ 허용 |
| 온라인 시험형 (카카오, 네이버 공채) | ❌ 브라우저 전용 |
| 오프라인 시험 (삼성 SW역량) | ❌ 불허 |

**Q. 테스트케이스 실행 가능한가?**
- 브라우저 에디터에서 예제 실행 및 커스텀 입력 ✅
- `print()`로 중간값 출력 → 실행 결과창에 함께 표시됨 ✅
- 로컬 파일 / 외부 툴 ❌

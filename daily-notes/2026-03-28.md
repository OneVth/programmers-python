## 🎯 핵심 학습 내용

### 1. 삼항 조건 표현식 — flag에 따라 다른 값 반환하기

```python
# flag에 따라 덧셈/뺄셈 분기
return (a + b) if flag else (a - b)
```

- Python의 삼항 표현식은 `값 if 조건 else 값` 형태
- `flag * (a+b) + (1-flag) * (a-b)` 같은 수식 트릭보다 가독성이 우선

```python
def solution_v1(a: int, b: int, flag: bool) -> int:
    return (a + b) if flag else (a - b)
```

### 2. 딕셔너리 매핑 vs if-elif 분기 — 조건 문자열

```python
# v1: if-elif 체인 (각 조건이 명확히 드러남)
if ineq == ">" and eq == "=":
    return int(n >= m)
elif ineq == "<" and eq == "=":
    ...

# v2: 딕셔너리 매핑 (분기를 데이터로 표현)
ops = {">=": n >= m, "<=": n <= m, ">!": n > m, "<!": n < m}
return int(ops[ineq + eq])
```

- `ineq + eq`로 두 문자를 합쳐 단일 키로 활용
- **주의:** 딕셔너리 방식은 값이 즉시 평가됨 → 비용이 큰 연산이라면 `lambda`로 lazy evaluation
  ```python
  ops = {">=": lambda: n >= m, ...}
  return int(ops[ineq + eq]())
  ```
- 조합의 수가 늘어날수록 딕셔너리 방식이 유지보수에 유리

### 3. `range` step으로 수열 생성 + `sum()` — 홀짝에 따라 다른 값 반환하기

```python
# 홀수 수열: 1, 3, 5, ..., n
range(1, n + 1, 2)

# 짝수 수열: 2, 4, 6, ..., n
range(2, n + 1, 2)
```

- `sum(i for i in range(...))` → `sum(range(...))` 로 더 간결하게 가능 (range는 iterable)
- 제너레이터 표현식은 리스트를 메모리에 올리지 않아 O(1) 공간

```python
def solution_v1(n: int) -> int:
    return (
        sum(i**2 for i in range(2, n + 1, 2))
        if n % 2 == 0
        else sum(i for i in range(1, n + 1, 2))
    )
```

**수학 공식으로 O(1) 풀이도 가능:**
- 홀수 합: `((n+1)//2) ** 2`
- 짝수 제곱합: `n * (n + 2) * (n + 1) // 6`

## 📝 오늘의 패턴 요약

```
val_a if condition else val_b          삼항 표현식 — 단순 분기의 표준
dict[key_a + key_b]                   두 조건 조합을 단일 키로 딕셔너리 매핑
sum(range(start, n+1, 2))             step range로 홀수/짝수 수열 합산
sum(expr for x in iterable)           제너레이터 합산 — 리스트 생성 없이 O(1) 공간
```

**공통 핵심:** 조건 분기가 고정된 조합일 때는 딕셔너리 매핑으로 코드를 데이터화하면 확장에 유리하다.
단, 딕셔너리의 값은 **즉시 평가**됨을 항상 유의할 것.

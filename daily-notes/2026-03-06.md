## 🎯 핵심 학습 내용

### 1. translate/maketrans vs lower+replace (문제 #181874)

```python
# "a"와 "A"만 대문자로, 나머지는 소문자로
# 방법 1: lower() + replace()
myString.lower().replace("a", "A")

# 방법 2: translate + maketrans (범용적)
myString.translate(str.maketrans("abcABC", "AbcAbc"))
```

**lower+replace가 나은 경우:** 변환 규칙이 단순할 때 (1~2개 문자)
**translate가 나은 경우:** 변환 매핑이 복잡할 때 (여러 문자 동시 치환)

### 3. math.prod() (문제 #181879)

```python
from math import prod

prod([1, 2, 3, 4])  # 24 (Python 3.8+)

# Python 3.7 이하에서는
from functools import reduce
import operator
reduce(operator.mul, [1, 2, 3, 4])
```

## 💡 오늘의 깨달음

1. **문자열 변환 체이닝**: `lower().replace()` 같은 체이닝은 Python 문자열이 불변이기에 가능 — 각 단계가 새 문자열 반환
2. **`int(bool)` 변환**: `True → 1`, `False → 0`으로 깔끔하게 정수 반환
3. **`math.prod`는 `sum`의 곱셈 버전**: Python 3.8에서 추가된 내장 함수로, reduce+mul 패턴을 대체

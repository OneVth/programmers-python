### 1. 병렬 리스트 누적 vs 딕셔너리 — 창고 정리 (#250126)

```python
def solution_v1(storage: list[str], num: list[int]) -> str:
    clean_storage = []
    clean_num = []
    for i in range(len(storage)):
        if storage[i] in clean_storage:
            pos = clean_storage.index(storage[i])
            clean_num[pos] += num[i]
        else:
            clean_storage.append(storage[i])
            clean_num.append(num[i])

    max_num = max(clean_num)
    return clean_storage[clean_num.index(max_num)]
```

- **C++ 원본 버그**: `clean_storage[num_item] = to_string(num[i])` → `storage[i]`를 저장해야 할 자리에 개수를 문자열로 변환해 넣는 실수
- 병렬 리스트 방식은 `in` + `index` 각 O(n) → 전체 **O(n²)**
- Python에서는 `dict` 또는 `Counter`로 O(n) 해결 가능:

```python
from collections import Counter
counter = Counter()
for item, cnt in zip(storage, num):
    counter[item] += cnt
return max(counter, key=counter.get)
```

### 2. 모드별 함수 분리 + dispatch — 가습기 (#250127)

```python
def solution_v1(mode_type: str, humidity: int, val_set: int) -> int:
    def func1(humidity, val_set):   # target: 설정값 기준
        return 3 if humidity < val_set else 1

    def func2(humidity):            # auto: 10단위 구간
        if humidity >= 50: return 0
        elif humidity >= 40: return 1
        elif humidity >= 30: return 2
        elif humidity >= 20: return 3
        elif humidity >= 10: return 4
        else: return 5

    def func3(humidity, val_set):   # minimum: 설정값 기준
        return 1 if humidity < val_set else 0

    dispatch = {"auto": lambda: func2(humidity),
                "target": lambda: func1(humidity, val_set),
                "minimum": lambda: func3(humidity, val_set)}
    return dispatch[mode_type]()
```

- C++ 빈칸 채우기 원본 구조를 그대로 Python 내부 함수로 옮긴 방식
- `func2`의 `if/elif` 체인은 수식으로도 표현 가능: `max(0, 5 - humidity // 10)` (단, humidity ≥ 50 클램핑 필요)
- **dict dispatch 패턴**: `if/elif` 대신 딕셔너리로 분기 → 모드 추가 시 확장성↑

### 3. 1-indexed 변환 실수 — 가채점 (#250128)

```python
def solution_v1(numbers: list[int], our_score: list[int], score_list: list[int]) -> list[str]:
    answer = []
    for i in range(len(numbers)):
        if our_score[i] == score_list[numbers[i] - 1]:
            answer.append("Same")
        else:
            answer.append("Different")
    return answer
```

- **C++ 원본 버그**: `score_list[i]` → 순회 인덱스로 조회 (학생 번호 무시)
- 수정: `score_list[numbers[i] - 1]` → 학생 번호(1-based)를 0-indexed로 변환
- list comprehension 버전:
  ```python
  return ["Same" if our_score[i] == score_list[numbers[i]-1] else "Different"
          for i in range(len(numbers))]
  ```

## 📝 오늘의 패턴 요약

```
병렬 리스트 (clean_storage + clean_num)   O(n²) — dict/Counter로 O(n) 대체 가능
내부 함수 + dict dispatch                 모드/분기 로직 분리, 확장성↑
score_list[numbers[i] - 1]               1-indexed 학생번호 → 0-indexed 배열 접근
```

**공통 패턴 — PCCE 디버깅 문제:**
버그는 주로 두 가지: ① 저장해야 할 값을 잘못된 값으로 대입, ② 인덱스 참조 오류 (순회 인덱스 vs 의미 인덱스).
코드를 읽을 때 각 변수가 "순서"인지 "번호(의미)"인지 구분하는 습관이 핵심.

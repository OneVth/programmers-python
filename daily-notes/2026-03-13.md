## 🎯 핵심 학습 내용

### 1. 닫힌 구간 → Python 슬라이싱 변환 (문제 #181895)

```python
# 닫힌 구간 [a, b]는 양 끝 포함 → Python은 반열린 구간 [a, b+1)
arr[a:b+1]  # 인덱스 a부터 b까지 (b 포함)

# 두 구간 연결하기
intervals = [[1, 3], [0, 4]]
arr[intervals[0][0]:intervals[0][1]+1] + arr[intervals[1][0]:intervals[1][1]+1]
```

### 2. itertools.chain 활용 (문제 #181895)

```python
from itertools import chain

# 방법 1: chain(*iterables) - 개별 전달
list(chain(list1, list2, list3))

# 방법 2: chain.from_iterable() - 이터러블의 이터러블
list(chain.from_iterable(arr[a:b+1] for a, b in intervals))
```

**chain vs + 연산자:**
| 방식 | 중간 리스트 생성 | 메모리 효율 |
|------|----------------|-------------|
| `list1 + list2` | ✅ 매번 새 리스트 | 낮음 |
| `+=` / `extend` | ❌ in-place 확장 | 중간 |
| `chain` | ❌ 이터레이터 연결 | 높음 |

### 3. 조건 탐색 패턴 비교 (문제 #181896)

```python
# 방법 1: range + 인덱스 접근 (작성한 풀이)
for i in range(len(num_list)):
    if num_list[i] < 0:
        return i
return -1

# 방법 2: enumerate (더 Pythonic)
for i, v in enumerate(num_list):
    if v < 0:
        return i
return -1

# 방법 3: next + 제너레이터 (한 줄)
next((i for i, v in enumerate(num_list) if v < 0), -1)
```

### 4. 슬라이싱 파라미터 조합 (문제 #181897)

```python
# 슬라이싱 기본 형태: list[start:stop:step]
# - start 생략 = 처음부터 (0)
# - stop 생략 = 끝까지 (len)
# - step 생략 = 1씩

# n에 따른 슬라이싱 매핑
n = 1: num_list[:b+1]      # 0 ~ b
n = 2: num_list[a:]        # a ~ 끝
n = 3: num_list[a:b+1]     # a ~ b
n = 4: num_list[a:b+1:c]   # a ~ b, c 간격
```

**통합 접근법 (고급):**
```python
# 모든 케이스를 파라미터 매핑으로 처리
params = {
    1: (None, b+1, None),
    2: (a, None, None),
    3: (a, b+1, None),
    4: (a, b+1, c)
}
start, stop, step = params[n]
return num_list[start:stop:step]
```

## 💡 오늘의 깨달음

1. **itertools 활용**: `chain.from_iterable()`은 제너레이터와 결합하면 메모리 효율적인 리스트 연결 가능 — 대용량 데이터에서 유용

2. **슬라이싱 연속 학습 6일차**: 닫힌 구간 변환, step 파라미터, 조건 분기까지 — 슬라이싱의 다양한 활용 패턴 정리 완료

3. **Pythonic 패턴**: `for i in range(len(x))`보다 `enumerate`가 더 읽기 좋고, `next() + 제너레이터`는 조건 탐색을 한 줄로 표현 가능

4. **None의 의미**: 슬라이싱에서 `None`은 "기본값 사용"을 의미 → `list[None:None:None]` = `list[:]`

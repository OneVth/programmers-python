## 🎯 핵심 학습 내용

### 1. 연산자 우선순위 함정 — 공배수 (#181936)

```python
# ❌ 버그: & 는 == 보다 우선순위가 높음
int(number % n == 0 & number % m == 0)
# 실제 파싱: int(number % n == (0 & number % m) == 0)
# → m 조건이 무시되고 n 조건만 검사됨

# ✅ 올바른 표현: 괄호로 먼저 bool 평가
int((number % n == 0) & (number % m == 0))
```

**연산자 우선순위:** `%` > `&` > `==`
`bool`의 비트 AND를 쓸 때는 반드시 `==` 비교를 괄호로 먼저 완료할 것.

---

### 2. LCM(최소공배수) 활용 — 공배수 (#181936)

```python
from math import lcm

# n과 m의 공배수 == lcm(n, m)의 배수
int(number % lcm(n, m) == 0)
```

- `lcm(a, b) = a * b // gcd(a, b)` — 내부적으로 유클리드 호제법 사용
- 여러 약수를 동시에 검사할 때 확장: `math.lcm(*ns)` (Python 3.9+)
- 시간복잡도: O(log(min(n, m))) vs 단순 나머지 두 번 O(1)

---

### 3. `max()` 동점 처리 — 두 수의 연산값 비교하기 (#181938)

```python
# "같으면 a⊕b 반환" 조건을 max() 인자 순서로 자연스럽게 처리
max(int(str(a) + str(b)), 2 * a * b)   # ✅ 같을 때 첫 번째(a⊕b) 반환
max(2 * a * b, int(str(a) + str(b)))   # ❌ 같을 때 2*a*b 반환
```

Python `max()`는 동점 시 **첫 번째 인자**를 반환한다.
인자 순서가 정확성에 영향을 미치는 경우를 인식할 것.

---

### 4. f-string vs `str()` — 더 크게 합치기 (#181939)

```python
str(a) + str(b)   # 명시적: 변환 → 연결 두 단계
f"{a}{b}"         # 암묵적: __format__ 호출, 한 표현식에 압축
```

- 단순 정수 변환에서는 성능 차이 없음
- 포맷 지정이 필요할 때 f-string이 유리: `f"{a:04d}"`

---

### 5. 문자열 `*` 연산자 — 문자열 곱하기 (#181940)

```python
my_string * k   # 가장 빠름: 최종 길이 미리 계산 후 메모리 한 번 할당
"".join([my_string] * k)   # 리스트 생성 오버헤드 있음
```

- 반복 연결(`+=`)은 매번 새 객체 생성 → O(n²), `*` 연산자는 O(n×k)

---

### 6. `"".join()` 패턴 — 문자 리스트를 문자열로 변환하기 (#181941)

```python
"".join(arr)        # 구분자 없이 이어 붙이기
",".join(arr)       # CSV 형태
"-".join(arr)       # 하이픈 구분
```

- `join`은 내부적으로 최종 길이를 미리 계산해 메모리를 한 번만 할당 → O(n)
- 인자는 문자열 iterable이면 무엇이든 가능 (list, tuple, generator, ...)

---

### 7. `zip()` + generator — 문자열 섞기 (#181942)

```python
# v1: 인덱스 기반
for i in range(len(str1)):
    answer.append(str1[i])
    answer.append(str2[i])

# v2: zip 기반 (더 Pythonic)
"".join(a + b for a, b in zip(str1, str2))
```

**generator vs list comprehension in `join`:**

```python
"".join(a + b for a, b in zip(...))    # generator: lazy evaluation
"".join([a + b for a, b in zip(...)])  # list: 즉시 평가
```

`join`은 내부적으로 iterable을 두 번 순회하므로, generator를 넘기면 실제로는 리스트로 변환된다.
큰 데이터에서는 리스트 컴프리헨션이 더 예측 가능한 성능을 낸다.

## 📝 오늘의 패턴 요약

```
int(num % n == 0)                          bool → int 변환으로 삼항 대체
int((a == 0) & (b == 0))                   비트 AND 쓸 때 괄호 필수 (우선순위 주의)
int(number % lcm(n, m) == 0)               공배수 판별을 LCM 한 번으로 처리
max(preferred, other)                      동점 시 첫 번째 반환 — 순서가 결과에 영향
int(str(a) + str(b))  ==  int(f"{a}{b}")   두 표현은 동일, f-string이 포맷 확장에 유리
my_string * k                              문자열 반복은 * 연산자가 가장 효율적
"".join(iterable)                          리스트 → 문자열 변환의 표준 패턴
zip(s1, s2)                                길이 같은 두 시퀀스 동시 순회 (인덱스 불필요)
```

**공통 핵심:** Python의 연산자 우선순위는 직관과 다를 수 있다.
특히 `&`, `|` 같은 비트 연산자와 `==` 비교 연산자가 섞일 때 반드시 괄호로 의도를 명시할 것.

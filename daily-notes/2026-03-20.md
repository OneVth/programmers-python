## 🎯 핵심 학습 내용

### 1. 접미사 배열 생성 패턴 (문제 #181909)

```python
# v1: 명시적 루프
suffix = []
for i in range(len(my_string)):
    suffix.append(my_string[i:])
return sorted(suffix)

# v2: 제너레이터 one-liner
return sorted(my_string[i:] for i in range(len(my_string)))
```

- 길이 n인 문자열의 접미사 총 길이: n + (n-1) + ... + 1 = n(n+1)/2 → **공간 O(n²)**
- `sorted()`는 이터러블을 받으므로 제너레이터를 직접 소비 가능 → 중간 리스트 생략
- 시간복잡도: 접미사 생성 O(n²) + 정렬 O(n² log n) = **O(n² log n)**

### 2. 음수 슬라이싱 (문제 #181910)

```python
my_string[-n:]   # 뒤의 n글자
my_string[:n]    # 앞의 n글자 (181907과 대칭)
```

| 패턴 | 의미 | 시간 | 공간 |
|------|------|------|------|
| `[:n]` | 앞의 n글자 (접두사) | O(n) | O(n) |
| `[-n:]` | 뒤의 n글자 (접미사) | O(n) | O(n) |
| `[s:e+1]` | 인덱스 s~e 구간 | O(e-s) | O(e-s) |

- Python 슬라이싱은 항상 **새 문자열 객체** 생성 (문자열 불변 특성)
- 제한 조건에서 `1 ≤ n ≤ len`이 보장될 때 경계 처리 불필요

### 3. 두 리스트 병렬 순회 패턴 (문제 #181911)

```python
# v1: range 인덱스 — verbose
my_strings[i][parts[i][0]:parts[i][1]+1]

# v2: enumerate + 구조 분해 — s, e 명시적 분리
for i, (s, e) in enumerate(parts):
    my_strings[i][s:e+1]

# v3: zip + 제너레이터 — Python 관용 패턴
"".join(x[y[0]:y[1]+1] for x, y in zip(my_strings, parts))
```

- `zip(a, b)`는 두 이터러블을 쌍으로 묶어 인덱스 변수 없이 병렬 순회
- v1 → v2 → v3 리팩토링은 **"필요한 변수만 노출"** 원칙의 점진적 적용

### 4. 부분 문자열 추출 + 변환 + 필터 패턴 (문제 #181912)

```python
# ✅ 중간 변수로 변환 1회
for e in intStrs:
    token = int(e[s:s+l])
    if token > k:
        answer.append(token)

# ⚠️ 리스트 컴프리헨션 — 변환 2회 (anti-pattern)
[int(e[s:s+l]) for e in intStrs if int(e[s:s+l]) > k]
```

- 조건부 컴프리헨션에서 **조건과 값에 동일한 계산이 중복**될 경우 명시적 루프가 유리
- walrus operator(`:=`)로 해결 가능: `[v for e in intStrs if (v := int(e[s:s+l])) > k]`

### 5. 문자열 구간 역전 패턴 (문제 #181913)

```python
# 방법 1: 문자열 재조합 (매 쿼리마다 새 문자열 생성)
answer = answer[:s] + answer[s:e+1][::-1] + answer[e+1:]

# 방법 2: list 변환 후 in-place 슬라이스 할당
arr = list(my_string)
for s, e in queries:
    arr[s:e+1] = arr[s:e+1][::-1]
return "".join(arr)
```

| 방법 | 매 쿼리당 객체 생성 수 | 특징 |
|------|----------------------|------|
| 문자열 재조합 | 슬라이스 3개 + 연결 1개 = 4개 | 직관적 |
| list in-place | 슬라이스 1개 (재할당) | 중간 문자열 객체 없음 |

- `str(my_string)`은 이미 문자열이므로 `list(my_string)`이 리스트화 목적에 더 명확
- q개의 쿼리, 길이 n인 문자열: **O(q × n)** 시간복잡도

## 📝 오늘의 패턴 요약

오늘 5문제는 전날에 이어 **문자열 슬라이싱의 심화 응용**이었다.

```
my_string[i:]          접미사 생성 (인덱스 i부터 끝)
my_string[-n:]         뒤의 n글자
my_string[s:e+1]       양 끝 포함 구간 (end+1 주의)
my_string[s:s+l]       길이 l짜리 부분 문자열
answer[s:e+1][::-1]    구간 역전
zip(a, b)              두 리스트 병렬 순회
```

전날(181907~181908)의 `[:n]` / `[-n:]` 패턴이 오늘(181909~181910)에서 **접미사 배열**과 **뒤 n글자**로 확장되었고,
전날(181905)의 구간 역전이 오늘(181913)에서 **다중 쿼리**로 반복 적용되는 점진적 난이도 구성이 인상적이었다.

## 🎯 핵심 학습 내용

### 1. 논리 연산자 직접 표현 (문제 #181917)

```python
return (x1 or x2) and (x3 or x4)
```

- Python `and`/`or`는 수학의 ∧/∨와 동일한 **우선순위**: `and > or`
- **단락 평가(Short-circuit evaluation)**: `False and ...` → 두 번째 피연산자 평가 안 함
- `bool or bool` → 결과가 항상 `bool`이므로 반환 타입 `-> bool` 정확함
- 드 모르간 법칙: `(A or B)` ≡ `not (not A and not B)` → NAND 게이트 구현에 활용

### 2. 단조 스택 (Monotonic Stack) (문제 #181918)

```python
# v1: 문제 조건 그대로 시뮬레이션 (while + i 수동 관리)
i = 0
while i < len(arr):
    if not stk or stk[-1] < arr[i]:
        stk.append(arr[i]); i += 1
    else:
        stk.pop()

# v2: 표준 단조 스택 패턴 (for + 내부 while)
for x in arr:
    while stk and stk[-1] >= x:
        stk.pop()
    stk.append(x)
```

- **v1 vs v2**: v1은 명세 기반 번역, v2는 패턴 인식 후 재구성 — 결과 동일, v2가 더 범용적
- **Amortized O(n)**: `pop()`은 `append()` 횟수 이하로만 발생 → 전체 연산 최대 2n번
- 단조 스택은 "다음 더 큰/작은 원소 찾기", "히스토그램 최대 직사각형" 등 다양한 문제에 응용

### 3. 콜라츠 수열 + fence-post 패턴 (문제 #181919)

```python
answer = []
while n > 1:
    answer.append(n)
    n = n // 2 if n % 2 == 0 else 3 * n + 1
answer.append(n)   # 마지막 1 추가 (fence-post)
return answer
```

- **Fence-post 문제**: 루프 조건 `n > 1`로 빠져나오면 n=1이 미추가 상태 → 루프 밖에서 append
- `while n > 1` vs `while n != 1`: 전자는 n=1 입력에도 안전하게 `[1]` 반환
- 시간복잡도는 **n의 함수로 예측 불가** — 입력 크기와 연산 횟수 간 불규칙한 패턴이 콜라츠 추측의 핵심

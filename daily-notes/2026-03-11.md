## 🎯 핵심 학습 내용

### 1. 슬라이싱 vs 인덱스 추적 (문제 #181893)

```python
# v1: 매번 새 리스트 생성 — O(Q * N)
for i, x in enumerate(query):
    if i % 2 == 0:
        answer = answer[: x + 1]
    else:
        answer = answer[x:]

# v2: 인덱스만 추적, 마지막에 한 번 슬라이싱 — O(Q + K)
start, end = 0, len(arr)
for i, x in enumerate(query):
    if i % 2 == 0:
        end = start + x + 1  # 상대 → 절대 인덱스 변환
    else:
        start += x
return arr[start:end]
```

**핵심 포인트:**
- 슬라이싱은 매번 O(K) 복사가 발생 → 반복 안에서 쓰면 비용 누적
- 인덱스 추적은 정수 연산만 하므로 O(1) → 반복 안에서 최적
- `end = start + x + 1`에서 `start`를 더하는 이유: `x`는 현재 잘린 배열 내의 **상대 인덱스**이므로 원본 기준 **절대 인덱스**로 변환 필요

### 2. 첫/마지막 위치 찾기 패턴 (문제 #181894)

```python
# 방법 1: index + pop (작성한 풀이)
first = arr.index(2)
answer = arr[first:]
while answer[-1] != 2:
    answer.pop()

# 방법 2: index + 역방향 index
first = arr.index(2)
last = len(arr) - 1 - arr[::-1].index(2)
return arr[first:last + 1]
```

| 방식 | 장점 | 단점 |
|------|------|------|
| `pop()` | 직관적, 추가 복사 없음 | 뒤에서 2가 멀면 pop 반복 |
| `[::-1].index()` | 한 줄로 간결 | 전체 리스트 뒤집기 복사 발생 |

### 3. 1-based → 0-based 변환 (문제 #181892)

프로그래머스 문제에서 "n번째"는 보통 1-based → Python에서는 `n-1`로 변환
```python
num_list[n - 1:]  # n번째 원소"부터" = 인덱스 n-1부터 끝까지
```

## 🎯 핵심 학습 내용

### 1. early return 패턴 (문제 #181890)

```python
def solution(str_list: list[str]) -> list[str]:
    for i, c in enumerate(str_list):
        if c == "l":
            return str_list[:i]
        elif c == "r":
            return str_list[i + 1:]
    return []
```

**"먼저 나오는 것을 찾아라" = early return이 최적:**
- `index()`를 쓰면 "l"과 "r" 각각 찾고 비교해야 함 + 없을 때 예외 처리 필요
- `enumerate` + early return은 한 번 순회로 해결, 예외 처리 불필요

### 2. 리스트 로테이션 (문제 #181891)

```python
# 방법 1: extend (과정을 명시)
answer = []
answer.extend(num_list[n:])
answer.extend(num_list[:n])

# 방법 2: 슬라이싱 연결 (결과를 선언)
num_list[n:] + num_list[:n]

# 방법 3: deque.rotate (in-place, 대용량에서 효율적)
from collections import deque
d = deque(num_list)
d.rotate(-n)
```

| 방식 | 스타일 | 새 리스트 생성 |
|------|--------|:----------:|
| extend | 명령형 (절차) | O |
| `+` 연결 | 선언형 (Pythonic) | O |
| deque.rotate | in-place | X |

### 3. 슬라이싱은 안전하다

```python
lst = [1, 2, 3]
lst[:100]  # [1, 2, 3] — 범위 초과해도 에러 없음
lst[100:]  # []         — 빈 리스트 반환
```

인덱싱(`lst[100]`)은 IndexError를 발생시키지만, 슬라이싱은 자동으로 범위를 조정함

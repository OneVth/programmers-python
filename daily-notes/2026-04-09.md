## 문제별 풀이 요약

### #340202 - 물 부족

**핵심 아이디어:** 매달 전 달 사용량에 변화율 적용 → 누적 사용량이 storage 초과 시 해당 인덱스 반환

```python
def solution_v1(storage: int, usage: int, change: list[int]) -> int:
    total_usage = 0
    for i, rate in enumerate(change):
        usage += int(usage * rate / 100)
        total_usage += usage
        if total_usage > storage:
            return i
    return -1
```

**버그 포인트 (디버깅 유형):** C++ 원본 코드의 9번째 줄

```cpp
// ❌ 원본 (버그)
usage = total_usage * change[i] / 100;

// ✅ 수정
usage += usage * change[i] / 100;
```

두 가지 오류가 동시에 존재:
1. **`total_usage` → `usage`**: 누적 사용량이 아닌 이전 달 사용량 기준으로 변화율 적용
2. **`=` → `+=`**: 변화량만 계산해서 더해야 하는데 전체를 덮어씀

## 오늘의 핵심 패턴

### `int()` vs `//` — 소수점 버림 방식의 차이

"소수점 이하를 버린다"는 표현은 **0 방향 버림 (truncation)** = C++ 정수 나눗셈과 동일.

| 표현식 | 동작 | -53.9에 적용 |
|--------|------|-------------|
| `int(-53.9)` | 0 방향 버림 (truncation) | **-53** |
| `math.floor(-53.9)` | 음수 방향 내림 (floor) | **-54** |
| `-5390 // 100` (Python) | floor division = floor | **-54** |

→ Python에서 "소수점 이하를 버린다"는 조건은 `//`가 아닌 `int()`를 써야 함.

### `int()` 위치에 따른 결과 차이

같아 보이는 두 표현식이 다른 결과를 낼 수 있음:

```python
usage, rate = 544, -10

# 방법 A: 전체를 int로 감싸기
int(usage * (1 + rate / 100))  # int(544 * 0.9) = int(489.6) = 489

# 방법 B: 변화량에만 int 적용 후 더하기
usage + int(usage * rate / 100)  # 544 + int(-54.4) = 544 + (-54) = 490
```

→ 문제의 의도(C++ 정수 나눗셈 동작)에 맞는 방식: **방법 B**

### `enumerate()` 활용

```python
# Before
for i in range(len(change)):
    rate = change[i]

# After
for i, rate in enumerate(change):
    ...
```

인덱스와 값을 동시에 쓸 때는 `enumerate()` 사용이 Pythonic.

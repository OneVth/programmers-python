## 🎯 핵심 학습 내용

### 1. range()의 반개구간 (문제 #181920)

```python
return list(range(start_num, end_num + 1))
```

- `range(a, b)`는 **b 미포함** — `end_num`까지 포함하려면 `+1` 필요
- 이 "반개구간(half-open interval)" 패턴은 슬라이싱 `lst[a:b]`, `enumerate`, `zip` 전반에 일관되게 적용
- `len(range(a, b)) == b - a`가 항상 성립 → 연속 구간을 `range(0,5), range(5,10)`처럼 자연스럽게 이어 붙일 수 있음

### 2. 집합 연산 3가지 (문제 #181921)

```python
# v1: 두 경우를 OR로 나열 (명시적이나 장황)
{"0", "5"} == set(str(i)) or {"5"} == set(str(i))

# v2: 부분집합 연산자 — "모든 원소가 허용 집합 안에 있다"
set(str(i)) <= {"0", "5"}

# v3: 차집합이 공집합 — "허용되지 않는 원소가 없다"
not set(str(i)) - {"0", "5"}
```

- 세 표현은 모두 동치이지만 사고 방향이 다름
- **v2가 가장 직관적**: 포함 관계를 자연어와 가장 가깝게 표현
- v3는 **부정적 사고방향** — "없음을 확인"하는 패턴, 화이트리스트보다 블랙리스트 검사에 어울림

### 3. 구간 쿼리 + 튜플 언패킹 (문제 #181922)

```python
# 인덱스 분해 (v1)
s = query[0]; e = query[1]; k = query[2]

# 튜플 언패킹 (v2) — 더 간결하고 의미 명확
for s, e, k in queries:
    for i in range(s, e + 1):
        if i % k == 0:
            arr[i] += 1
```

- `for s, e, k in queries` 패턴은 `enumerate`, `zip`, `dict.items()`와 동일한 언패킹 문법
- 쿼리 조건이 **"인덱스 기준"** (이전 문제: 값 기준)임에 주의 — `arr[i]`가 아닌 `i`가 k의 배수

### 4. bisect_right vs 선형 탐색 (문제 #181923)

```python
# v1: 정렬 + 이진탐색 — O(n log n) per query
sorted_arr = sorted(arr[s:e+1])
idx = bisect.bisect_right(sorted_arr, k)
answer.append(-1 if idx >= len(sorted_arr) else sorted_arr[idx])

# v2: 필터 + min — O(n) per query (더 효율적)
bigger = [i for i in arr[s:e+1] if i > k]
answer.append(-1 if not bigger else min(bigger))
```

- **`q * n` < `q * n log n`** → v2가 점근적으로 더 빠름
- v1(bisect)의 장점은 **정렬을 재활용**할 때 드러남 — 매 쿼리마다 슬라이싱 후 새로 정렬하면 오히려 비효율
- `bisect_right(arr, k)`: k 이하의 마지막 원소 **다음** 인덱스 반환 → `arr[idx]`가 k 초과 최솟값

### 5. Python 다중 대입 스왑 (문제 #181924)

```python
answer = list(arr)  # 원본 복사 — side effect 방지
for i, j in queries:
    answer[i], answer[j] = answer[j], answer[i]  # 임시 변수 불필요
```

- Python 다중 대입은 **오른쪽을 먼저 튜플로 평가** 후 왼쪽에 언패킹 → `tmp` 변수 없이 스왑 가능
- `list(arr)` 복사가 중요 — 원본을 수정하면 호출자 데이터 변경(side effect) 발생
- 시간복잡도 O(q): 각 쿼리가 O(1) 스왑이므로 쿼리 수에만 비례

## 📝 오늘의 패턴 요약

오늘 5문제는 **기초 수열 생성**, **집합 연산**, **구간 쿼리** 시리즈(3, 2, 4번)로 구성됐다.

```
range(s, e + 1)                     반개구간 — 끝 포함 시 +1
set(str(i)) <= {"0", "5"}           부분집합 연산자로 자릿수 필터링
for s, e, k in queries:             튜플 언패킹으로 쿼리 분해
min(v for v in arr[s:e+1] if v>k)  필터 + min 조합 (bisect보다 단순하고 빠름)
a[i], a[j] = a[j], a[i]            Python 다중 대입 스왑
```

같은 "수열과 구간 쿼리" 시리즈라도 조건이 **인덱스 기준(#4)** vs **값 기준(#2)** vs **스왑(#3)**으로
각각 달랐다 — 문제 번호에 현혹되지 말고 조건을 꼼꼼히 읽는 것이 핵심.

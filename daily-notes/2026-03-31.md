## 🎯 핵심 학습 내용

### 1. 내장 메서드로 한 줄 해결 — 대소문자 바꿔서 출력하기 (#181949)

```python
def solution_v1(str: str) -> str:
    return str.swapcase()
```

- `str.swapcase()` — 대문자 → 소문자, 소문자 → 대문자로 일괄 변환, O(n)
- 직접 구현 시: `isupper()` / `islower()` 분기 또는 `ord()` / `chr()` ASCII 연산 활용
- **파라미터명 `str`이 내장 타입 `str`을 shadowing** → 함수 내부에서 `str()` 호출 불가
  → 실무에서는 `s`, `text` 등 다른 이름 사용 권장 (프로그래머스 지정 이름이라 예외)

### 2. 문자열 반복 연산자 — 문자열 반복해서 출력하기 (#181950)

```python
def solution_v1(str: str, n: int) -> str:
    return str * n
```

- `str * n` — 파이썬 문자열 반복 연산자, O(len(str) × n)
- 내부적으로 **최종 길이를 먼저 계산 후 메모리를 한 번에 할당** → 루프 `+=` 방식보다 효율적
- n이 상수(≤5)로 제한 → 실질적으로 O(len(str))로 볼 수 있음

```python
# 비교: 루프 방식은 매 반복마다 새 객체 생성
result = ""
for _ in range(n):
    result += str   # O(n²) — 피해야 할 패턴
```

### 3. f-string 멀티라인 출력 — a와 b 출력하기 (#181951)

```python
def solution_v1(a: int, b: int) -> str:
    return f"a = {a}\nb = {b}"
```

- `\n`을 f-string 안에 직접 삽입해 멀티라인 문자열 반환
- `print()` 대신 `return`으로 구현 → runner.py의 단순 비교 테스트와 호환
  - `print()`는 stdout 캡처 필요, `return`은 값 비교만으로 검증 가능

```python
# print 방식 (stdout 캡처 필요)
print(f"a = {a}")
print(f"b = {b}")

# return 방식 (테스트 호환, 권장)
return f"a = {a}\nb = {b}"
```

### 4. 참조 반환과 불변성 — 문자열 출력하기 (#181952)

```python
def solution_v1(str: str) -> str:
    return str
```

- `return str`은 새 문자열을 복사하지 않고 **참조만 반환** → O(1)
- Python 문자열 불변성: 호출자가 반환값을 수정하려 하면 그때서야 새 문자열 생성
- 최대 1,000,000글자 입력에도 성능 문제 없는 이유

## 📝 오늘의 패턴 요약

```
str.swapcase()                 대소문자 일괄 반전 — 내장 메서드 활용
str * n                        문자열 반복 — 한 번에 메모리 할당, 루프 += 보다 효율적
f"a = {a}\nb = {b}"           f-string 안 \n으로 멀티라인 반환
return str                     참조 반환 O(1) — 복사 아님
```

**공통 주의:** 파라미터명으로 `str`, `list`, `int` 등 내장 타입명을 사용하면 **shadowing** 발생.
해당 함수 내에서 내장 타입/함수를 호출할 수 없게 됨 → 실무 코드에서는 반드시 피할 것.

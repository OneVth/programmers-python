## 문제별 풀이 요약

### #340200 - 닉네임 규칙

**핵심 아이디어:** 문자 치환 규칙 순차 적용 후 길이 조건(4~8자) 보정

```python
for c in nickname:
    if c == "l":   answer += "I"
    elif c == "w": answer += "vv"
    elif c == "W": answer += "VV"
    elif c == "O": answer += "0"
    else:          answer += c
while len(answer) < 4:
    answer += "o"
if len(answer) > 8:
    answer = answer[:8]
```

**버그 포인트 (디버깅 유형):** C++ 원본 코드에서 두 가지 오류 존재
1. `if` → `while`: 조건을 한 번만 확인하면 길이가 1~2인 입력에서 패딩 부족
2. `< 3` → `< 4`: 경계값 오류 (off-by-one), 길이가 정확히 3일 때 패딩 누락

**배운 점:** `while len(answer) < 4: answer += "o"` 는 Python 관용 표현으론 `answer.ljust(4, 'o')` 한 줄로 대체 가능

### #340201 - 버스

**핵심 아이디어:** 정거장별 승/하차 카운트 누적 → 남은 좌석 = `max(0, seat - num_passenger)`

```python
def solution_v1(seat, passengers):
    def func3(station):  # Off 카운트
        return sum(1 for p in station if p == "Off")
    def func4(station):  # On 카운트
        return sum(1 for p in station if p == "On")

    num_passenger = 0
    for station in passengers:
        num_passenger += func4(station)
        num_passenger -= func3(station)
    return func1(seat - num_passenger)  # max(0, x)
```

**빈칸 채우기 분석:** C++ 헬퍼 함수 4개의 역할 파악이 핵심
- `func1`: `max(0, x)` — 음수 방지
- `func2`: `min(0, x)` — 양수 방지 (이 문제에서 미사용)
- `func3`: "Off" 개수 카운트
- `func4`: "On" 개수 카운트

**배운 점:**
- C++ 헬퍼를 Python 중첩 함수(nested function)로 이식 → 네임스페이스 오염 방지
- `func1(seat - func1(num_passenger))`에서 안쪽 `func1`은 불필요 (승객 수는 항상 ≥ 0)
- Python 관용 표현: `max(0, seat - num_passenger)` 한 줄로 동일 효과

## 오늘의 핵심 패턴

| `if` vs `while` 패딩 | 닉네임 규칙 | 조건 만족 시까지 반복 → `while`, 한 번만 → `if` |
| off-by-one 경계값 | 닉네임 규칙 | `< 3` vs `< 4`, 경계 포함 여부 항상 확인 |

| `max(0, x)` | 버스 | 음수 방지 관용 표현, `func1` 역할과 동일 |
| 중첩 함수(nested function) | 버스 | 스코프 제한, 외부 네임스페이스 오염 방지 |

### 1. split() vs split(' ') 동작 차이 (문제 #181868)

**문제:** 공백 한 개 이상으로 구분된 문자열에서 단어를 순서대로 배열로 추출

```python
def solution_v1(my_string: str) -> list[str]:
    return my_string.split()
```

**핵심 인사이트: Python은 두 가지를 완전히 다른 알고리즘으로 처리한다**

```python
s = "  i   love  you  "

s.split()      # ['i', 'love', 'you']
s.split(' ')   # ['', '', 'i', '', '', 'love', '', 'you', '', '']
```

| 특성 | `split()` | `split(' ')` |
|------|-----------|--------------|
| 앞뒤 공백 | 자동 제거 | 빈 문자열 생성 |
| 연속 공백 | 하나의 구분자로 취급 | 공백마다 분리 |
| 결과에 `''` | 절대 포함 안 됨 | 포함될 수 있음 |
| 대상 공백 | 모든 whitespace (`\t`, `\n` 등) | 정확히 `' '`만 |

**CPython 내부:** `STRINGLIB(split_whitespace)`와 `STRINGLIB(split_char)` 두 가지 C 함수로 별도 구현

### 2. split() 활용 패턴 정리

```python
# 패턴 1: 공백 정규화
"  hello   world  "
" ".join(s.split())  # "hello world"

# 패턴 2: maxsplit으로 분리 횟수 제한
"name age city".split(maxsplit=1)  # ['name', 'age city']

# 패턴 3: split(' ')로 같은 결과를 얻으려면 (비효율)
[x for x in s.strip().split(' ') if x]  # strip + split + 필터 필요
```

## 💡 오늘의 깨달음

1. **`split()` 무인자 호출은 별도 알고리즘**: 단순히 `split(' ')`의 축약이 아니라 완전히 다른 C 함수로 구현됨
2. **공백 문제의 정답 패턴**: 프로그래머스 공백 관련 문제에서는 `split()`이 거의 항상 최적해
3. **`" ".join(s.split())`**: 연속 공백을 단일 공백으로 정규화하는 관용 표현

## 🎯 핵심 학습 내용

### 1. 역방향 딕셔너리 매핑 (문제 #181925, #181926)

```python
# 수 조작하기 1 (순방향): 문자 → 델타값
tab = {"w": 1, "s": -1, "d": 10, "a": -10}

# 수 조작하기 2 (역방향): 델타값 → 문자
tab = {1: "w", -1: "s", 10: "d", -10: "a"}
```

- 두 문제는 **서로 역방향 관계** — forward 딕셔너리를 뒤집으면 reverse 딕셔너리가 됨
- `if/elif` 체인 대신 딕셔너리를 쓰면 조건 추가 시 딕셔너리 항목만 추가하면 됨 (개방-폐쇄 원칙)
- 181925(2번)의 핵심: `numLog[i] - numLog[i-1]`이 키, 해당 문자가 값

### 2. Python 반복 성능 비교 (문제 #181926)

```python
# 방법 1: for 루프 누적 — 2.65s (1000회, n=100,000)
for c in control:
    answer += tab[c]

# 방법 2: sum(generator) — 3.13s (가장 느림)
n + sum(tab[c] for c in control)

# 방법 3: sum(map) — 1.42s (가장 빠름)
n + sum(map(tab.__getitem__, control))
```

- **`sum(generator)`가 for보다 느린 이유**: 제너레이터는 매 yield마다 Python 프레임 전환 오버헤드 발생
- **`sum(map(...))`이 가장 빠른 이유**: `map()`과 `dict.__getitem__`이 모두 C 레벨 함수 → 전체 루프가 C에서 실행
- 코딩 테스트 규모(n ≤ 100,000)에서는 세 방식 모두 충분히 빠름 — 가독성 우선

### 3. 음수 인덱싱과 복사 (문제 #181927)

```python
def solution_v1(num_list: list[int]) -> list[int]:
    answer = list(num_list)          # 원본 복사
    if num_list[-1] > num_list[-2]:  # 음수 인덱싱으로 끝에서 접근
        answer.append(num_list[-1] - num_list[-2])
    else:
        answer.append(num_list[-1] * 2)
    return answer
```

- `num_list[-1]`, `num_list[-2]`: 길이 계산 없이 마지막 두 원소 참조
- `list(num_list)` 복사: 원본 보호(side effect 방지) — 함수형 관점에서 중요한 습관
- 삼항 표현식 대안: `num_list + [a - b if a > b else a * 2]`

### 4. 문자열 이어붙이기로 수 합산 (문제 #181928)

```python
odd, even = [], []
for i in num_list:
    if i % 2 == 0:
        even.append(str(i))
    else:
        odd.append(str(i))
return int("".join(odd)) + int("".join(even))
```

- **숫자 이어붙이기 = 문자열 연산**: `int("".join(str(n) for n in nums))` 패턴
- 루프 한 번에 `odd`, `even` 두 리스트를 채우는 방식 → O(n) 1회 순회
- `filter()`를 두 번 쓰면 순회 2회 발생 — 명시성 vs 효율성 트레이드오프

## 📝 오늘의 패턴 요약

오늘 4문제는 **수 조작하기** 시리즈(1, 2번)와 리스트 조작 문제들로 구성됐다.

```
{delta: char}                        역방향 딕셔너리 — forward를 뒤집으면 reverse
sum(map(dict.__getitem__, iterable)) C 레벨 최적화 — generator보다 빠름
num_list[-1], num_list[-2]           음수 인덱싱 — 끝에서 접근
list(original)                       복사본 생성 — side effect 방지
int("".join(str(n) for n in nums))   숫자 이어붙이기 관용 패턴
```

**수 조작하기 1, 2**는 같은 데이터를 방향만 바꿔 다루는 대칭적 문제 쌍이었다.
동일한 딕셔너리 구조에서 키와 값의 역할만 뒤집으면 두 문제를 모두 해결할 수 있다.

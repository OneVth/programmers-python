# set의 내부 구조 — 해시 테이블과 O(1) 탐색

> Python의 set은 해시 테이블로 구현되어 있어 `in` 연산이 O(1)이다

## 핵심 원리

```python
exclude = {1, 16, 6, 15, 0, 10, 11, 3}

# "6이 있는가?" 를 확인할 때
hash(6)  # → 6 (정수는 자기 자신이 해시값)
index = hash(6) % table_size  # → 바로 해당 슬롯으로 점프
```

리스트처럼 처음부터 순서대로 찾는 게 아니라, **해시값으로 저장 위치를 바로 계산**한다.
마치 책의 목차에서 페이지 번호를 보고 바로 펼치는 것과 같다.

---

## 내부 메모리 구조

```
테이블 크기: 8 (기본)

슬롯[0] → 0   (hash(0) % 8 = 0)
슬롯[1] → 1   (hash(1) % 8 = 1)
슬롯[2] → 10  (hash(10) % 8 = 2)
슬롯[3] → 3   (hash(3) % 8 = 3)
슬롯[4] → (비어있음)
슬롯[5] → (비어있음)
슬롯[6] → 6   (hash(6) % 8 = 6)
슬롯[7] → 15  (hash(15) % 8 = 7)
```

각 원소는 해시값에 따라 **고정된 슬롯**에 저장된다. 탐색 시 해당 슬롯만 확인하면 되므로 O(1)이다.

---

## 해시 충돌 (Collision)

두 값의 해시가 같은 슬롯을 가리킬 수 있다.

```python
# hash(1) % 8 = 1
# hash(9) % 8 = 1  ← 충돌!
```

Python은 **오픈 어드레싱(Open Addressing)** 방식으로 해결한다.
충돌 시 다음 빈 슬롯을 찾아 저장한다 (단순 +1이 아닌 프로빙 공식 사용).

```
충돌 시 다음 슬롯 계산:
next_index = (5 * current_index + 1 + perturb) % table_size
perturb >>= 5
```

이 때문에 **최악의 경우** (모든 값이 충돌) O(n)이 될 수 있지만,
Python의 해시 함수가 충돌을 잘 분산시키므로 **평균적으로 O(1)** 이다.

---

## 리사이징

테이블이 2/3 이상 차면 자동으로 크기를 늘린다.

```
원소 수 → 테이블 크기
0~5     → 8
6~10    → 16
11~21   → 32
...        (약 2배씩 확장)
```

리사이징 자체는 O(n)이지만, 빈도가 낮아 **amortized O(1)** 이 유지된다.

## list vs set 비교

```
list:  [1, 16, 6, 15, 0, 10, 11, 3]
       → "6 있어?" → 1 확인, 16 확인, 6 발견! (3번 비교)
       → "99 있어?" → 전부 확인 (8번 비교) → 없음

set:   {1, 16, 6, 15, 0, 10, 11, 3}
       → "6 있어?" → 슬롯[6] 확인 → 있음! (1번 비교)
       → "99 있어?" → 슬롯[3] 확인 → 없음! (1번 비교)
```

| 연산 | list | set |
|------|------|-----|
| `in` 탐색 | O(n) 순차 | O(1) 해시 |
| 삽입 | O(1) append | O(1) 평균 |
| 삭제 | O(n) 탐색+이동 | O(1) 평균 |
| 순서 보장 | O | X |
| 중복 허용 | O | X |

## 해시 가능 조건

set에 저장하려면 **해시 가능(hashable)** 해야 한다.

```python
# ✅ 해시 가능: 불변(immutable) 타입
{1, 2, 3}           # int
{"a", "b"}          # str
{(1, 2), (3, 4)}    # tuple

# ❌ 해시 불가: 가변(mutable) 타입
{[1, 2]}            # TypeError: unhashable type: 'list'
{{1: 2}}            # TypeError: unhashable type: 'dict'
```

내부 값이 변하면 해시값도 바뀌어 슬롯 위치를 찾을 수 없기 때문이다.

---

## 코딩테스트 활용 패턴

```python
# ❌ 리스트에서 반복 탐색 — O(n * m)
for x in data:
    if x in some_list:
        ...

# ✅ set 변환 후 탐색 — O(n + m)
some_set = set(some_list)
for x in data:
    if x in some_set:
        ...
```

**반복적인 `in` 연산이 필요할 때는 항상 set 변환을 먼저 고려하라.**

---

## 요약

1. **해시 함수**로 값의 저장 위치를 계산 → 바로 접근 (O(1))
2. **해시 충돌** 시 오픈 어드레싱으로 다음 슬롯 탐색
3. 테이블이 2/3 차면 **리사이징** (amortized O(1) 유지)
4. **순서 보장 X**, **해시 불가능한 값(list, dict) 저장 불가**
5. 반복적 `in` 연산이 필요하면 **list → set 변환**이 핵심 최적화

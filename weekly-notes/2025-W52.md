## 💡 이번 주 핵심 패턴 정리

### 1. In-place vs 새 배열 Trade-off (181830)

```python
# v1: In-place 수정 - O(1) 추가 공간
def solution_inplace(arr):
    rows, cols = len(arr), len(arr[0])
    if rows > cols:
        for row in arr:
            row.extend([0] * (rows - cols))
    elif cols > rows:
        arr.extend([[0] * cols for _ in range(cols - rows)])
    return arr

# v2: 새 배열 생성 - O(m²) 공간, 더 안전
def solution_new(arr):
    n = max(len(arr), len(arr[0]))
    answer = [[0] * n for _ in range(n)]
    for i in range(len(arr)):
        for j in range(len(arr[0])):
            answer[i][j] = arr[i][j]
    return answer
```

| | In-place | 새 배열 |
|---|---|---|
| Space | O(1) 추가 | O(m²) |
| 원본 보존 | ❌ 수정됨 | ✅ 유지 |
| 코딩 테스트 | 부작용 주의 | **안전하고 권장** |

### 2. 대칭 행렬 판별 — 루프 최적화 (181831)

```python
# v1: 상삼각 영역만 검사 - O(n²/2), O(1) ← 최적
for i in range(n):
    for j in range(i + 1, n):  # 핵심: i+1부터 시작
        if arr[i][j] != arr[j][i]:
            return 0
return 1

# v2: 전치행렬 비교 - O(n²), O(n²)
return int(arr == list(map(list, zip(*arr))))
```

### 3. 진짜 최적화 vs 가짜 최적화

```python
# ❌ 가짜 최적화 - 루프 횟수 동일!
for i in range(n):
    for j in range(n):      # 여전히 n² 번 반복
        if i >= j:
            continue        # 조건 검사만 추가 (오버헤드)

# ✅ 진짜 최적화 - 루프 범위 자체를 줄임
for i in range(n):
    for j in range(i + 1, n):  # 루프 자체가 n(n-1)/2 번!
```

**성능 차이 (n=100):**
| 방식 | 루프 횟수 | 실측 시간 |
|------|-----------|-----------|
| 전체 비교 | 10,000 | 0.32s |
| continue | 10,000 + α | 0.28s |
| **range 조정** | **4,950** | **0.15s** ← 2배 빠름! |

### 4. zip(*arr) — 전치행렬 생성

```python
arr = [[1, 2, 3],
       [4, 5, 6]]

# Step 1: *arr로 언패킹
# zip([1,2,3], [4,5,6])

# Step 2: zip이 같은 인덱스끼리 묶음
# (1, 4), (2, 5), (3, 6)

# Step 3: list로 변환
list(map(list, zip(*arr)))
# [[1, 4], [2, 5], [3, 6]]
```

**대칭 행렬의 성질:** A == Aᵀ (전치해도 자기 자신)

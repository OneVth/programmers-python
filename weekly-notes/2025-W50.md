### 1. 120812 최빈값 — 인덱스 혼동

```python
# ❌ 내가 쓴 코드
return mc[0][1]  # 빈도수를 반환

# ✅ 올바른 코드
return mc[0][0]  # 요소를 반환
```

**원인**: `Counter.most_common()` 반환값 `[(요소, 빈도), ...]`에서 인덱스 혼동

**교훈**: 튜플 언패킹 형태를 명확히 기억하기

### 2. 120846 합성수 — range와 sqrt 오류

```python
# ❌ 내가 쓴 코드
for i in range(4, n):           # n 미포함
    for j in range(2, int(n**0.5) + 1):  # n의 제곱근 사용

# ✅ 올바른 코드
for i in range(4, n + 1):       # n 포함
    for j in range(2, int(i**0.5) + 1):  # i의 제곱근 사용
```

**원인**:
1. `range(4, n)`은 n을 포함하지 않음 → `range(4, n+1)`
2. 각 숫자의 약수를 찾을 때는 그 숫자(`i`)의 제곱근까지만 확인

**교훈**: range 경계와 변수 스코프 꼼꼼히 확인

### 3. 120848 팩토리얼 — bisect 함수 선택

```python
# ❌ 내가 쓴 코드
bisect_left(factorials, n) - 1

# ✅ 올바른 코드
bisect_right(factorials, n) - 1
```

**원인**: `i! ≤ n`을 만족하는 최대 i를 찾을 때
- `bisect_left`: n과 같은 값의 **왼쪽** 위치 → n이 정확히 팩토리얼일 때 틀림
- `bisect_right`: n과 같은 값의 **오른쪽** 위치 → -1하면 정확히 해당 인덱스

**교훈**: "이하(≤)"는 `bisect_right - 1`, "미만(<)"은 `bisect_left - 1`

## 💡 새로 배운 것

### 유클리드 호제법에 max/min이 필요 없는 이유

```python
def gcd(a, b):
    while b != 0:
        a, b = b, a % b
    return a
```

- `a < b`일 때: `a % b = a`이므로 첫 반복에서 자동으로 swap
- 예: `gcd(3, 12)` → `(12, 3)` → `(3, 0)` → `3`
